## 执行要求

### 必须严格遵守的规则

1. **四步串行流程，不得跳过 Step 1（校验政府官网）**：
   - 每个区县必须先从 `data/gov-website-links.ts` 读取政府官网 URL
   - 用 `fetch_webpage` 验证可达性 + 地名匹配 + 是否为门户主站
   - **DNS 失败 / 连接超时 / 地名不匹配 → 标记 gov-suspicious，调用 gov-site-finder 修复后才能继续**
   - 不得在官网可疑状态下直接找预决算链接

2. **遇到政府官网失效时的处理**：
   - 读取 `.github/skills/gov-site-finder/SKILL.md`（特别是 G2 方法：从市级门户找县级官网）
   - 更新 `data/gov-website-links.ts` 中的错误域名
   - 用修复后的新 URL 重新执行 Step 1
   - 如确实无法修复，记录为 `gov-dns-fail`

3. **subagent 的 prompt 必须内联核心规则**：
   - 不要只写"参考 SKILL.md"，因为 subagent 往往只读前 100 行
   - 将 SKILL.md 并行策略节的 subagent 提示词模板（全文）直接粘入 prompt
   - 明确告知 subagent：DNS/连接失败时必须标记 gov-suspicious 而非 "not-found"

4. **同市路径模式复用**：
   - 在一个市的第一个区县找到有效路径后，立即在同市其他区县尝试相同模式
   - 减少不必要的逐站探测

5. **真并行 subagent**：
   - 将 3-4 个 subagent 的 `runSubagent` 调用放在同一个 `<function_calls>` block 中
   - 每轮完成后汇总，处理 gov-suspicious 项，再发下一轮
   - 不要串行一个一个发 subagent

### 经验教训（来自鲁豫陕甘青藏六省 + 云贵黑吉辽五省）

1. **域名严重不一致是常态，不是例外**：
   - 陕西有 7 个县域名与 gov-website-links.ts 记录不符（长安区 changan→changanqu, 鄠邑区 huyi→xahy, 华州区 hzq→hzqu, 镇巴县 zhenba→zb, 府谷县 fugu→fg, 兴平市 xingping→snxingping, 武功县 sxwg→wugong）
   - 甘肃有 13 个、河南有 5 个、西藏有 9 个域名需要修正
   - **遇到 DNS 失败时，第一反应是"域名可能变了"，而不是"网站不存在"**

2. **统一 CMS 的路径复用效率极高**（10 倍）：
   - 宝鸡 12 县全部使用 col/col 系统，找到 1 个就能批量推 11 个
   - 铜川 3 县统一 resources/site/{N} 模式
   - 延安 13 县全部使用市级门户集中公开页面 `/gk/fdzdgknr/czxx/czyjs/xq/1.html`
   - 定西 7 县统一 col/col 模式
   - 天水 7 县统一 `.htm` 后缀
   - 陇南 9 县统一 CMS column 系统参数化 URL
   - 渭南 8 县统一 `/zfxxgk/fdzdgknr/czxx/` 路径
   - **先做一个市的第一个县，确认模式后批量验证其余县**

3. **市级集中公开平台可能代替县级独立页面**：
   - 延安 13 县全部指向市级门户的 `/xq/` 路径（县区级筛选器）
   - 遇到县级子域名 DNS 失败时，检查市级门户是否有集中的县区预决算公开栏目

4. **Subagent 拆分策略（防超时）**：
   - 每个 subagent 控制在 10-15 个县以内
   - 22+ 个县的批次容易超时（本次陕西 Batch1 和 Batch4 都超时了）
   - 湖南 113 个缺口建议拆成 8-10 个 subagent:
     - 湖北 23 个缺口拆 2-3 个
     - 湖南按城市分组，每组 10-15 个

5. **JS 渲染 CMS 站点**：
   - HTTP 探测拿到空骨架，必须用 `fetch_webpage`
   - GUID 路径不可猜测，需从导航菜单或搜索引擎获取
   - 超过 2 轮同类失败就换策略

6. **搜索引擎辅助很有用**：
   - 直接访问失败时，Baidu 搜索 `site:xxx.gov.cn 预决算` 可发现正确路径
   - 宜君县、眉县 等都是通过 Baidu 找到的
   - 但 Baidu 有时触发验证码，需多次尝试

7. **慢站 ≠ 死站，西部/中部站要耐心**：
   - 首次超时不代表不可达，至少重试 2 次
   - 502/503 是临时故障
   - SSL 错误时先试 HTTP

8. **每省的政府官网修复要同步做**：
   - 不要把 gov-suspicious 放到最后统一处理
   - 每完成一个 subagent 批次，立即处理返回的 suspicious 项

9. **安康/商洛等站点使用 Node-{id} 或 category-{id} 模式**：
   - 这类 CMS 的 URL 含动态 ID，不可猜测
   - 必须从导航页面提取或搜索引擎索引获取

10. **陕西各市 CMS 差异巨大**（同省不同 CMS）：
    - 西安各县独立路径、铜川 resources 模式、宝鸡 col 模式、咸阳 zfxxgk 模式、渭南统一路径、延安市级集中、汉中 shtml 系统、安康 Node/category 系统、商洛 .htm 系统
    - **不要假设同省=同 CMS，每市需独立探测**
